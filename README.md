# VPN Bot for 3X-UI

Telegram-бот для управления VPN-подключениями через панель [3X-UI](https://github.com/MHSanaei/3x-ui). Позволяет создавать, редактировать и удалять клиентские подключения, автоматически контролировать трафик и скорость, а также отслеживать статус всех клиентов — прямо из Telegram.

## Возможности

- **Управление клиентами** — создание, редактирование, удаление подключений через Telegram
- **VLESS-ключи и QR-коды** — генерация актуальных ключей подключения с динамическим определением адреса сервера
- **Мониторинг трафика** — автоматическая проверка использования трафика каждые N минут
- **Ступенчатое ограничение скорости** — автоматическое снижение скорости при достижении 80% и 95% лимита трафика (через Linux `tc`)
- **Статус-панель** — сводка по всем клиентам с возможностью закрепления в чате
- **Управление операторами** — администратор может добавлять других операторов бота
- **Управление банами** — просмотр и разбан IP-адресов (fail2ban)
- **Ежемесячный сброс** — автоматический сброс счётчиков трафика в заданный день

## Требования

- **ОС:** Linux (Ubuntu/Debian) — необходим `iproute2` для ограничения скорости
- **Python:** 3.10+
- **3X-UI панель** — установленная и настроенная с VLESS+Reality inbound
- **Telegram Bot Token** — получить через [@BotFather](https://t.me/BotFather)

## Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/cmepthuklolka1/vpn-bot.git /opt/vpn-bot
cd /opt/vpn-bot
```

### 2. Настроить конфигурацию

```bash
cp config.example.json config.json
nano config.json
```

Заполните обязательные поля:

| Поле | Описание |
|------|----------|
| `telegram.bot_token` | Токен Telegram-бота от @BotFather |
| `telegram.admin_id` | Ваш Telegram ID (узнать: @userinfobot) |
| `panel.url` | URL панели 3X-UI (например `https://IP:PORT`) |
| `panel.base_path` | Базовый путь панели |
| `panel.username` | Логин от панели |
| `panel.password` | Пароль от панели |
| `panel.inbound_id` | ID inbound в панели (обычно `1`) |
| `domain` | (Необязательно) Домен/IP сервера для ключей. Если пустой — определяется автоматически из настроек inbound |

### 3. Запустить установку

```bash
sudo bash install.sh
```

Скрипт автоматически:
- Установит системные зависимости
- Создаст виртуальное окружение Python
- Установит Python-пакеты
- Настроит systemd-сервис с автозапуском

### 4. Запустить бота

```bash
sudo systemctl start vpn-bot
```

## Обновление

```bash
cd /opt/vpn-bot
sudo bash deploy.sh
```

Скрипт обновит код из GitHub, установит зависимости и перезапустит бота. Конфигурация и база данных не затрагиваются.

## Миграция с ручной установки

Если бот уже работает на сервере без Git:

```bash
# 1. Сохранить данные
cd /opt/vpn-bot
cp config.json config.json.backup
cp bot_data.db bot_data.db.backup

# 2. Остановить бота
sudo systemctl stop vpn-bot

# 3. Подключить Git
git init
git remote add origin https://github.com/cmepthuklolka1/vpn-bot.git
git fetch origin
git checkout -b main origin/main --force

# 4. Восстановить данные
cp config.json.backup config.json
cp bot_data.db.backup bot_data.db

# 5. Обновить зависимости и запустить
source venv/bin/activate && pip install -r requirements.txt
deactivate
sudo systemctl start vpn-bot
```

База данных полностью совместима — существующие клиенты сохранятся.

## Конфигурация скорости

Бот автоматически ограничивает скорость клиента при приближении к лимиту трафика:

| Порог | Параметр | По умолчанию | Действие |
|-------|----------|-------------|----------|
| < 80% | `speed_base_mbps` | 0 (без лимита) | Полная скорость |
| 80–95% | `speed_80pct_mbps` | 10 Мбит/с | Умеренное ограничение |
| > 95% | `speed_95pct_mbps` | 1 Мбит/с | Сильное ограничение |

Лимиты настраиваются индивидуально для каждого клиента через бота.

## Настройка 3X-UI: логи доступа

Для отображения количества подключённых устройств в статусе бота необходимо включить логи доступа в панели 3X-UI:

1. Перейдите в **Настройки Xray → Основное → Логи**
2. **Логи доступа** — установите `./access.log` (вместо `none`)
3. **Уровень логов** — рекомендуется `warning` или `info`
4. Нажмите **Сохранить** → **Перезапуск Xray**

Без этой настройки бот не сможет определить количество подключённых устройств — в статусе будет отображаться `📱 0[лимит]`.

## Настройка 3X-UI: Xray API (gRPC)

Если в панели используется функция **LIMIT_IP** (ограничение количества подключённых устройств), необходимо убедиться что Xray API работает корректно. Без него панель не сможет отключать лишние IP-адреса при превышении лимита.

1. Перейдите в **Настройки Xray → Расширенный шаблон**
2. Убедитесь что в секции `"inbounds"` есть блок API с протоколом `dokodemo-door`:

```json
{
  "tag": "api",
  "listen": "127.0.0.1",
  "port": 62789,
  "protocol": "dokodemo-door",
  "settings": {
    "address": "127.0.0.1"
  }
}
```

3. Если стоит `"protocol": "tunnel"` — замените на `"dokodemo-door"`
4. Нажмите **Сохранить** → **Перезапуск Xray**

> ⚠️ Это внутренний служебный протокол (слушает только на 127.0.0.1). Изменение не влияет на VPN-подключения, ключи и шифрование клиентов.

## Управление

```bash
# Статус сервиса
sudo systemctl status vpn-bot

# Логи systemd
sudo journalctl -u vpn-bot -f

# Логи бота
tail -f /opt/vpn-bot/bot.log

# Перезапуск
sudo systemctl restart vpn-bot

# Остановка
sudo systemctl stop vpn-bot

# Запуск
sudo systemctl start vpn-bot

# Включить автозапуск при старте системы
sudo systemctl enable vpn-bot

# Отключить автозапуск
sudo systemctl disable vpn-bot
```
